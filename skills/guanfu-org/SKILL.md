## 汇报关系

| 汇报人 | 汇报对象 | 说明 |
|--------|---------|------|
| 所有成员 | CEO（观复） | 常规汇报，任务进度 |
| 所有成员 | 阁主 | 重大事项、战略方向 |
| CTO/CFO/COO/CMO/CPO/CHRO | CEO | 部门汇报 |
| CSO+QA | CEO | 战略与质量汇报 |

## 任务派发流程

### 方式一：阁主直接派发（最高优先级）
```
阁主 → @具体成员 / @全体成员
```

### 方式二：CEO分配
```
阁主 → CEO（观复）→ 部门负责人 → 成员
```

### 方式三：部门内部流转
```
部门负责人 → 部门成员
```

### 方式四：跨部门协作
```
发起方 → CEO协调 → 接收方部门 → 执行成员
```

## 任务派发标准模板

当需要派发任务时，使用以下格式：

```
📋 任务名称：[任务标题]
👤 负责人：[名字]
📅 截止时间：[日期]
📝 任务描述：[详细说明]
🔗 相关资源：[链接/文件]
✅ 验收标准：[如何算完成]
```

## 使用场景

1. **查询组织架构** → 提供架构图和成员职责
2. **了解汇报关系** → 提供汇报表格
3. **任务派发** → 提供标准模板并提醒 @被派发人
4. **汇报流程咨询** → 提供汇报格式和时效要求
5. **跨部门协作** → 引导走CEO协调流程

## 重要原则

- **派发任务必须 @ 被派发人**（阁主教诲）
- 紧急任务直接 @ 相关成员
- 常规任务按汇报链传递
- 重大事项需同时汇报阁主

## 扩展参考资料

本技能提供以下扩展参考资料，位于 `references/` 目录：

| 文件 | 内容 | 使用场景 |
|------|------|----------|
| **templates.md** | 任务派发模板、汇报模板、会议纪要模板、请示模板 | 直接复制使用模板 |
| **roles.md** | 各角色详细职责说明书、部门协作关系 | 深入了解职责 |
| **flowcharts.md** | 6个核心流程图（任务派发、汇报、入职、项目启动、质量审核、跨部门协作） | 复杂场景参考 |
| **policies.md** | 公司制度汇总（考勤、报销、会议、决策、质量、信息安全、考核等） | 日常规范 |
| **onboarding.md** | 新成员入职指引、常见问题、联系方式 | 团队扩张时使用 |

### 使用方法

- 当需要**模板**时 → 读取 `references/templates.md`
- 当需要**详细职责**时 → 读取 `references/roles.md`
- 当需要**流程图**时 → 读取 `references/flowcharts.md`
- 当需要**制度规范**时 → 读取 `references/policies.md`
- 当有新成员加入时 → 读取 `references/onboarding.md`
